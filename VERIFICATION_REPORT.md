Сессия этапа 0, такт 2. [Историческая справка: на момент сдачи все файлы были `verified: false`; 11.06.2026 простановка выполнена по ADR-001 (auto-verify v1), см. verification-блоки в файлах.]
Канонической при расхождениях тело↔приложение принята **версия тела книги** (см. `content-packs/DUPLICATE_DIFFS.md`, 12 расхождений — все мелкая редактура, ни одного механического).

## Источник

В knowledge книга лежит как предобработанный бандл: постраничные JPEG 980×1232 + текстовый слой.
Извлечение шло из текстового слоя; руны (картинки-глифы) в нём потеряны и восстановлены
по позиционной конвенции вёрстки, подтверждённой визуально по сканам:

- **строчные feat-таблицы:** строка Ока — первая (без номера), затем 1–10, строка руны — последняя;
- **таблица преданий:** секция Ока — первая, секция руны — вторая, затем 1–10.

Численный канон (книга правил, «выполнение проверок»): **Око = 11, руна Гэндальфа = 12**.
В контент-файлах — только литералы `"eye"` / `"gandalf_rune"`, маппинг на числа — дело движка.

## Находки и решения, требующие твоего взгляда

1. **«6×6» подтверждено**: шесть покровителей — Балин (стр. 7), Бильбо, Кирдан, Гэндальф (стр. 8),
   Гильраэн, Том Бомбадил и леди Златеника (стр. 9). Балин в приложении есть (стр. 28),
   но в первом проходе легко теряется — он в хвосте страницы после таблицы сцен обнаружения.
2. ~~Дифф «достать/найти» у Бильбо-1~~ — снято ревью: в обоих источниках «найти»
   (grep «достать» по стр. 8 и 29 = 0); «достать» было ошибкой цитирования по памяти
   в примере схемы такта 1, в данные не попало (KNOWN_ISSUES F2, исправлено).
3. **Структурированные эффекты** (`effects` / `consequence`) проставлены только там, где книга даёт
   явную механику: ±Око в удаче/неудаче, последствия сцен путешествия и подвалы деталей сцен.
   Сверь знаки и значения — это то, что движок будет применять автоматически.
4. **Изнурение «—»** у «Вдохновляющего вида» закодировано как `null` (не 0): «баллы не
   начисляются» и «начисляется ноль» движок будет различать.
5. **«Отменить провал»** в особых успехах применим только в совместной игре — оставлен в таблице
   с пометкой в описании; движку соло-режима понадобится фильтр (задача этапа 1, не контента).
6. Манёвренная позиция, Советы, значимые встречи и пр. — **правила, не таблицы**: пойдут в
   `mechanics/` карточками в следующей сессии этапа 0.

## Воспроизводимость

```
cd tools/extraction
python3 build_solo_tables.py   # пересборка JSON из source_pages/
python3 validate.py            # схемы + структурные инварианты
```

Парсеры покрывают четыре регулярных семейства (предания, feat-таблицы, детали сцен,
покровители); одиночные таблицы (ответы, сцены путешествия, lookup'ы, параметры) — курируемые
конфиги в `build_solo_tables.py`, транскрибированные с текста/сканов. Это честная граница
автоматизации: для 32-страничной книги конфиг надёжнее эвристик.
