# ROADMAP_SESSIONS — план по сессиям до конца проекта

Канонический разбивочный план: этапы из `brodyazhnik-architecture-v1.md` §9 →
сессии (чаты) → деливераблы (подчаты). Живёт в репо (план в чате дрейфует, как и
память; этот файл — нет).

**Точность вперёд (честно):** ближнее (Stage 2–3) — детально и планируемо;
Stage 4 крупнее и грубее (зависит от исходов 2–3); Stage 5 эмерджентный (ведётся
твоими транскриптами), дроблю только по якорям. План **пересобирается на каждом
хендоффе**: выполненный чат отмечается, дальний этап переразбивается, если
реальность разошлась.

**Единицы:**
- **Чат N.x** = одна сессия: старт с zip+handoff, финиш с handoff перед
  насыщением контекста (сигнал 🔴 ПОРА В НОВЫЙ ЧАТ).
- **Подчат N.x.y** = один трёхтактный деливерабл: скелет → «го» → реализация+тесты+зип.

**Привязка стартера (см. архитектурный документ, гейт отложенного):** стартер
каждой сессии указывает, какой чат/подчат этого файла он закрывает, и эхо-выводит
открытые пункты `DEFERRED.md` с `DUE ≤ текущего этапа`. Этап не закрыть, пока такой
пункт открыт.

Статусы: ✅ done · ▶ in-progress · ⬜ pending.
