# Нарративный контракт `движок → Хранитель`

> Чат 2.1.a. Канон контракта между детерминированным движком и нарративным слоем.
> Тип: [`orchestrator/src/contract.ts`](../orchestrator/src/contract.ts) (ASCII,
> content-clean). Промпт-потребитель: [`prompts/keeper.system.v0.md`](../prompts/keeper.system.v0.md).
> Машинная проверка запрещёнки: [`evals/src/antislop.ts`](../evals/src/antislop.ts).
> Источник: `brodyazhnik-architecture-v1.md` sec 2.3.

## Идея

Движок катит всю механику и отдаёт **структурный пакет** (`NarrativePackage`).
Хранитель **рендерит прозу** из пакета и не трогает механику. Контракт — про то, что
именно отдаётся и что Хранителю **запрещено выдумывать**.

## Четыре жёстких правила (sec 2.3)

1. **Числа, кости, исходы — только из пакета.** Запрещено упоминать механику, которой
   в пакете нет.
2. **Результат оракула обязателен к вплетению** — конкретный и здешний. Второй уровень
   (`oracle.detail`) тоже, если пришёл. Недостающий оракул Хранитель **не бросает**.
3. **Чувства героя не описываются** (sec 0.9) — только воспринимаемое (звук/запах/
   осязание/видимое).
4. **Длина по типу сцены** — из `length_target`. Конкретные диапазоны — калибровочный
   параметр (источник §2.3.4; владелец `tone.md`/**LT1**, калибруется в 2.3), не в
   контракт-типе.

## Запрещено выдумывать (явный список)

| Категория | Запрет | Источник истины |
|---|---|---|
| Числа | Любая величина, которой нет в пакете (урон, ЦЧ, дельты трекеров) | `dice`, `patch` |
| Кости | Что выпало; перебрасывать или «уточнять» бросок | `dice` |
| Исходы | Успех/провал/степень — решает движок | `dice.outcome` |
| Оракулы | Подменять/досочинять результат оракула; катить недостающий уровень | `oracle` (+ `oracle.detail`, SD1) |
| Механика | Упоминать подсистему, которой нет в пакете (стойки, Око, Изнурение…) | весь пакет |
| Состояние | Заявлять изменение, которого нет в `patch` | `patch` |
| Лор | Противоречить `lore_chunks` или выдумывать факты мира поверх | `lore_chunks` |
| Непрерывность | Переписывать установленные факты (NPC, обещания) | `journal_facts` |

## Терминологический контроль

Кальки чужих систем запрещены (TOR — не D&D): «хиты», «спасбросок», «очки жизни»,
«класс брони», «мана», «прокачка» и т.п. Канон — термины Пандора Бокс. Детерминированный
сид этих запретов и generic-клише — `evals/src/antislop.ts` (расширяется в 2.3).
ВК-пастиш (стоп-лист сеттинга) — pack-side, `tone.md` (**DEFERRED LT1**), не в коде.

## Карта пакета (поле → смысл)

- `intent` — класс намерения (sec 5). Классификатор (отдельный вызов vs основной) —
  **отложено в чат 2.4** (open question Q1); поле классификатор-агностично.
- `scene` / `length_target` — тип сцены + рамки длины (числовые диапазоны калибруются в
  `tone.md`/**LT1**, источник §2.3.4; контракт несёт только форму).
- `dice` — Кость испытания (11=Око, 12=руна), Кости успеха, `outcome`.
- `oracle.result_ref` + `oracle.detail` — верхний уровень + второй уровень (**SD1**,
  катится движком в 2.2.a; слот в контракте уже есть, переделки не будет).
- `patch` — дельты трекеров и состояния (summary; Хранитель не пересчитывает).
- `lore_chunks` — opaque RAG-куски из пака (**LT1**: производство `lore/` — Stage 2).
- `journal_facts` — недавние факты для непрерывности (sec 2.4).

## Выход Хранителя

`KeeperOutput`: проза + опц. `questions`. Уточняющий вопрос — два режима (`free_text` /
`options`), **MVP — `free_text`** (open question Q2: «вероятно оба»). Вопрос задаётся,
когда выбор определяет следующий ход.
