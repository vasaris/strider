## 2. Архитектура движка (arch §2.1–2.2)
**State (§2.1):** `hero` / `world` / `scene` / `log`. Строй **ИНКРЕМЕНТАЛЬНО** — только срезы, нужные текущей подсистеме (не весь объект сразу). Ключевые формулы: ЦЧ соло = **18 − рейтинг характеристики**; рейтинг братства = **3 + бонусы**; порог Ока = **18/16/14 + модификаторы** по местности (Пограничные/Дикие/Тёмные).

## 3. Порядок сборки + первая веха (ПОДТВЕРЖДЁН Иваном)
Цепочка зависимостей journey-прохождения:
1. **Dice** — Кость испытания d12 (**11 = Око, 12 = руна Гэндальфа** — канон `common.schema.json $defs.featDieFace`), Кости успеха d6 (6 = знак успеха), благополучный/злополучный бросок, вдохновлённые проверки. Сидируемый RNG в основе. *Тесты:* распределения граней, воспроизводимость по сиду, благо/зло-логика.
2. **Checks** — ЦЧ = 18 − характеристика; степени риска (обычная/опасная/безрассудная → исход провала); особые успехи (траты знаков: узнать/бесшумно/быстро/влияние/преимущество). *Тесты:* формула ЦЧ, пороги степеней, маппинг особых успехов.
3. **Oracles** — таблица ответов (5 вероятностей; руна = «да+», Око = «нет+»), таблица преданий (Действие/Аспект/Фокус, 12×6), удача/неудача (триггер руна/Око). Чтение verified-таблиц из пака + детерминированный бросок+lookup. *Тесты:* распределения, маппинг руна/Око, ровно один default-likelihood.
4. **Eye of Mordor** — рост (+1 за Око вне боя, +N за баллы Тени, +2 за Око в таблице неудачи), порог по местности, сцена обнаружения при достижении, сброс после сцены / в Фазе братства. *Тесты:* каждое правило роста, срабатывание порога, сброс.
5. **Journey** — таблица сцен пути (бросок по местности) → 7 типов сцен → таблицы деталей → последствия/Изнурение → значимые встречи как эскалация. *Тесты:* покрытие таблицы, привязка `detail_table`, накопление Изнурения.

**ПЕРВАЯ ВЕХА (= критерий выхода Stage 1):** **CLI-прохождение путешествия на тестовом герое, без LLM, на фиксированном сиде, воспроизводимо end-to-end.** Тестовый герой — минимальный фикстур (культура/характеристики/навыки), достаточный для journey.

**ПОСЛЕ вехи** (нужны для полного завершения Stage 1, НЕ для первой вехи): **Бой** (тяжелейшая: стойки, манёвренная позиция, логика врагов, выход из боя), **Советы** (рейтинг сопротивления 3/6/9, N успехов до 3 провалов), **Прогрессия** (вехи → баллы, Йоль), **Фаза братства** (восстановление, снятие Тени по значимости 1/2/3, начинания). Их порядок уточним после journey-CLI.

## 4. Тест-дисциплина
Каждая подсистема — юнит-тесты на сидированном RNG (принцип №4):
- распределения таблиц совпадают с книжными (оракулы/сцены/события);
- формула ЦЧ считается верно; рост Ока — по правилам;
- воспроизводимость: один сид → один результат.
**Golden-сценарий journey** (фиксированный сид → детерминированная последовательность сцен) — аналог golden-transcripts; дифф при любом изменении движка.

## 5. Стек и раскладка (§8)
- **`engine/`** — отдельная TS-библиотека (чистый TS, ноль контента, сидируемый RNG). Юридически чист — единственная часть с возможным коммерческим будущим (§10); дисциплина разделения с первого коммита.
