# DEFERRED — канонический реестр отложенной работы

Единственный источник правды по работе, сознательно отложенной на будущие этапы.
Памяти и разрозненных `HANDOFF_*.md` недостаточно (дрейфуют между сессиями); этот
файл в репо — нет.

## Правило-гейт (см. brodyazhnik-architecture-v1.md, §9 roadmap)

- Критерий выхода этапа N **не выполнен**, пока в этом реестре открыт пункт с
  `DUE ≤ N`.
- Стартер каждой сессии этапа **обязан** эхо-вывести открытые пункты своего этапа.
- Новый отложенный пункт заносится сюда **в той же сессии**, что и решение отложить.
- Закрытие: пункт переносится в раздел «Закрыто» с коммитом, который его закрыл.

`DUE` — самый ранний этап/гейт, обязанный пункт закрыть. Привязанные к этапу
пункты также проставлены инлайн в roadmap-строках архитектурного документа, чтобы
их нельзя было не встретить при старте этапа.

## Открытые

### HC1 — Создание героя / жизненные пути
- **Статус:** open
- **DUE:** Этап 4 (онбординг); вероятно нужно в Этапе 3 (лист героя).
- **Что:** Движкового потребителя у создания персонажа нет. 8 верифицированных
  таблиц жизненных путей (Stage 0) не читаются ничем; герой собирается руками
  (тестовый). Нужен модуль: культура + призвание + броски по путям →
  стартовый `HeroState`.
- **Почему отложено:** Критерий выхода Этапа 1 («CLI-прохождение путешествия без
  LLM») явно «на тестовом герое» — генерация не требовалась. Онбординг документ
  относит к Этапу 4.
- **Где живёт:** таблицы — `content-packs/kv/` (lifepath); состояние —
  `engine/src/hero/state.ts`; модуля `engine/src/creation/` пока нет.
- **Блокирует:** «первое полное приключение» (Этап 4), если приключение должно
  начинаться с генерации героя, а не с готового.

### FV1 — Фаза первых залпов (combat)
- **Статус:** open
- **DUE:** Этап 4 (доска боя) — либо раньше, как только разыгрывается соло-дальний
  бой.
- **Что:** `runCombat` по явному скоупу melee-only; enum `CombatPhase.first_volleys`
  есть, драйвера фазы нет. Число залпов (`pre_melee_volleys`) — scene-determined
  (зависит от дистанции, фикс-таблицы в паке нет).
- **Почему отложено:** Tact 5b явно ограничил скоуп мили-раундами; первые залпы и
  внезапность вынесены за него.
- **Где живёт:** `engine/src/combat/runCombat.ts` (строка о скоупе),
  `engine/src/combat/types.ts` (`CombatPhase`); родитель манёвра —
  `content-packs/kv/mechanics/manevrennaya_poziciya.json` (`pre_melee_volleys`).
- **Блокирует:** полный дальний бой; pre-melee часть манёвренной позиции.
