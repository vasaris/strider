# Анти-слоп чек-лист Хранителя

> Чат 2.1.c — человекочитаемый чек-лист. Машинно-проверяемое ядро (стоп-листы как
> regex/словари) — [`evals/src/antislop.ts`](../evals/src/antislop.ts), расширяется в
> чате 2.3 вместе с LLM-судьёй и golden transcripts. Контракт — `docs/NARRATIVE_CONTRACT.md`.
> Источник протокола: `brodyazhnik-architecture-v1.md` sec 0.2 / sec 2.5.

Разделение: **deterministic** — ловится regex-ом в `evals/` (CI-гейт); **judge** —
оценивает LLM-судья по рубрике (чат 2.3); **human** — ручная вычитка транскриптов (Stage 5).

## A. Механика и контракт (deterministic + judge)
- [ ] Ни одного числа/исхода/кости вне входного пакета (`NARRATIVE_CONTRACT.md`).
- [ ] Не упомянута механика, которой нет в пакете.
- [ ] Результат оракула вплетён — конкретный, здешний; второй уровень тоже, если пришёл.
- [ ] Длина в рамках `length_target`.

## B. Терминология (deterministic — `evals/`)
- [ ] Нет калек чужих систем: «хиты», «спасбросок», «очки жизни», «класс брони»,
      «мана», «прокачка» → `block`.
- [ ] Термины — канон Пандора Бокс.

> **Граница seed (by design, не блокер):** regex ловит **точные формы** из словаря.
> Словоизменение (хиты/хитов/хитам…) и пропуски — добирает LLM-судья в 2.3; расширение
> словарей (морфология, новые кальки/клише) — задача чата 2.3.

## C. Слоп прозы (deterministic seed + judge)
- [ ] **Replacement Test**: фразу нельзя переставить в любой другой текст без потери.
- [ ] Нет generic-клише («время словно остановилось», «по спине пробежал холодок»,
      «воздух наполнился»…) → `warn`, переписать.
- [ ] **Ведём звуком / запахом / осязанием**, не зрением по умолчанию.
- [ ] Нет ВК-пастиша (стоп-лист сеттинга — `tone.md`, **LT1**; «древнее зло
      пробуждается» и т.п.).

## D. Точка зрения (judge — sec 0.9)
- [ ] Чувства героя не названы ярлыком; показаны через восприятие и тело.
- [ ] Нет всеведущего рассказчика («little did…», «герой не знал, что…»).

## E. Голоса культур (judge — чат 2.2.c)
- [ ] Хоббит ≠ гном ≠ дунэдайн; у персонажа одна устойчивая речевая привычка.

## F. Роль Хранителя (judge)
- [ ] Хранитель не комментирует механику вне входного пакета; не ломает сцену
      служебным текстом.
