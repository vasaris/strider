## 1. НЕРУШИМЫЕ ПРИНЦИПЫ (не пересматривать молча; видишь причину — предложи явно и жди подтверждения)

1. **Механика не живёт внутри LLM.** Броски/проверки/таблицы/состояние — детерминированный код; нарративный слой получает структурированные результаты, не выдумывает числа/исходы.
2. **Оракулы — механика движка.** Таблицы бросает rules engine; LLM интерпретирует выпавшее, не заменяет.
3. **engine/ не содержит контента.** Весь контент — в `content-packs/kv/` за манифестом.
4. **RNG сидируемый**, каждый бросок воспроизводим.
5. **Без `verified:true` (ставит только `mark_verified.py` после всех гейтов) в прод не грузится.**
6. **Контур приватный.** Никогда не предлагай публичные анонсы, открытые ссылки, монетизацию, демо.

- **Терминологический канон — перевод Пандора Бокс**: Хранитель, Кость испытания, Кости успеха, Изнурение, баллы Тени, Надежда, ЦЧ, благополучный/злополучный бросок, Бдительность Ока, Фаза братства.
- **Источники правды по убыванию**: (1) `brodyazhnik-architecture-v1.md`, (2) PDF книг (правило КВ/ИдО проверяется по PDF/сканам, **не по памяти**), (3) решения в диалогах.
- **Анти-паттерны**: ❌ правило по памяти без сверки; ❌ контент в engine/ «временно»; ❌ проза мимо стоп-листов; ❌ реализация/правка без подтверждённого скелета; ❌ молчком менять арх-решения; ❌ публикация/монетизация.

## 2. РАБОЧИЙ РЕЖИМ (как мы работаем — Ивану повторять не нужно)

- **Формат сообщений — два блока:** «**ДЛЯ ИВАНА**» (что/зачем + готовые к вставке **ASCII-команды терминала**, по шагам) и «**ДЛЯ CLAUDE CODE DESKTOP**» — сейчас всегда **ПУСТО** (ландим в терминале).
- **Делегирование Ивана (его постоянная установка):** «*как лучше, так и делаем, иди по /goal если можно, если нет — всегда выбираем наилучший с точки зрения итогового результата вариант*». То есть: **не переспрашивай по рутинным решениям** — выбирай наилучший для итога вариант и показывай готовый, проверенный результат. Пауза (stop-and-ask) — только на реальную двусмысленность, которую нельзя решить верно без него.
- **Стандарт 110/100.** Каждая сдача — с честной **самооценкой** и явным указанием слабых мест. Иван ждёт **pushback**, а не согласие.
- **`/goal`-дисциплина:** финишная линия = машинно-проверяемые гейты (без прилагательных, которые ревьюер не может проверить); одна цель на батч; «не трогать прошлые батчи»; turn cap. Determinism-финиш = `check_determinism.py` byte-identical. Файл `docs/GOAL_CONDITIONS.md` — ready-to-paste /goal-блоки.
- **Критичные ограничения вставки (научены на ошибках):**
  - **Кириллица в ТЕКСТЕ команды ломает вставку.** Никогда не пиши кириллицу внутри shell-команд; файлы/PDF ищи ASCII-маской.
  - **Большие heredoc'и обрезаются.** Поэтому большие файлы (скрипты, доки) отдавай **скачиваемым артефактом** (`present_files` → `/mnt/user-data/outputs/…`) + публикуй **sha256**; Иван кладёт файл и сверяет хеш.
  - **Коммиты — только флагами `-m`** (никогда `-F -` / heredoc). Сообщения коммитов — ASCII (транслит, не кириллица).
- **Проза (если будешь писать/править summary, few-shot):** анти-слоп — Replacement Test; рус-стоп-лист («атмосфера пронизана», «является», «данный», «поистине», «безусловно», «стоит отметить» и т.д.); Толкин-пастиш — главный риск; не описывать чувства героя, лидировать звуком/запахом/осязанием; голоса NPC по культурам (хоббит ≠ гном ≠ дунэдайн).
- **Язык** — русский по умолчанию; идентификаторы/код/коммиты — английский.
- **Стек зафиксирован** (Next.js App Router PWA, Tailwind, Postgres/Supabase, Anthropic API) — не предлагай замену без явного запроса. (К ревью не относится, но фон.)

## 5. ВЫХОД (mark_verified) + КООРДИНАЦИОННАЯ ОГОВОРКА

Механизм: `python3 tools/extraction/mark_verified.py --dir <dir> --gate2 "<2a+2b evidence>" --gate3 "<lynn evidence>"`. Он **перегоняет gate-1 прямо сейчас** (откажется, если красный), штампует `verified:true`, идемпотентен (уже-verified пропускает).

**Оговорка (важно):** `--dir content-packs/kv/mechanics` штампует **все 189** rule_card (КВ + ИдО) — они в одной папке и ни одна не verified, идемпотентность их не спасёт. Поэтому **140 КВ-карт должны ТОЖЕ пройти 2a/3** до запуска, иначе молча проштампуются непроверенные КВ-карты. Варианты: (а) провести 2a/3 по всему паку из 189 разом; либо (б) добавить в `mark_verified.py` фильтр по id/префиксу и штамповать только проверенное подмножество. Реши это с Иваном **до** запуска mark_verified.
