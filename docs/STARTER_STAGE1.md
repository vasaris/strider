## 1. НЕРУШИМЫЕ ПРИНЦИПЫ (не пересматривать молча; видишь причину — предложи явно и жди подтверждения)

1. **Механика не живёт внутри LLM.** Броски/проверки/таблицы/состояние — детерминированный код; нарративный слой (Stage 2) получает структурированные результаты, не выдумывает числа/исходы.
2. **Оракулы — механика движка.** Таблицы бросает rules engine; LLM интерпретирует выпавшее, не заменяет.
3. **`engine/` не содержит контента.** Ни строки ВК в коде движка; весь контент — в `content-packs/kv/` за манифестом. Числа правил НЕ хардкодить — читать из verified-JSON.
4. **RNG сидируемый**, каждый бросок воспроизводим (критично для тестов/evals).
5. **Без `verified:true` в прод не грузится.** Загрузчик движка отказывает, если `manifest.json` → `all_verified:false`.
6. **Контур приватный.** Никогда не предлагай публичные анонсы, открытые ссылки, монетизацию, демо. `engine/` юридически чист (ноль чужого текста) — дисциплина разделения с первого коммита.

- **Терминологический канон — перевод Пандора Бокс**: Хранитель, Кость испытания, Кости успеха, Изнурение, баллы Тени, Надежда, ЦЧ, благополучный/злополучный бросок, Бдительность Ока, Фаза братства.
- **Источники правды по убыванию**: (1) `brodyazhnik-architecture-v1.md`, (2) книга КВ/ИдО + verified-JSON пака (правило проверяется по ним, **не по памяти**), (3) решения в диалогах.
- **Анти-паттерны**: ❌ правило по памяти без сверки; ❌ контент/число книги в `engine/` «временно»; ❌ реализация без подтверждённого скелета; ❌ молчком менять арх-решения; ❌ побочные эффекты в чистых функциях движка; ❌ публикация/монетизация.

## 2. РАБОЧИЙ РЕЖИМ (как мы работаем — Ивану повторять не нужно)

- **Формат сообщений — два блока:** «**ДЛЯ ИВАНА**» (что/зачем + готовые к вставке **ASCII-команды терминала**, по шагам) и «**ДЛЯ CLAUDE CODE DESKTOP**» — сейчас всегда **ПУСТО** (ландим в терминале).
- **Делегирование Ивана (его постоянная установка):** «*как лучше, так и делаем, иди по /goal если можно, если нет — всегда выбираем наилучший с точки зрения итогового результата вариант*». Не переспрашивай по рутине — выбирай наилучший для итога вариант и показывай готовый, проверенный результат. Пауза (stop-and-ask) — только на реальную двусмысленность.
- **Стандарт 110/100.** Каждая сдача — с честной **самооценкой** и явным указанием слабых мест. Иван ждёт **pushback**, а не согласие.
- **`/goal`-дисциплина:** финишная линия = машинно-проверяемые тесты (без прилагательных, которые нельзя проверить); одна цель на батч; turn cap. `docs/GOAL_CONDITIONS.md` — ready-to-paste /goal-блоки.
- **Критичные ограничения вставки (научены на ошибках):**
  - **Кириллица в ТЕКСТЕ команды ломает вставку.** Никогда не пиши кириллицу внутри shell-команд; файлы/PDF ищи ASCII-маской.
  - **Большие heredoc'и обрезаются.** Большие файлы (модули, скрипты, доки) отдавай **скачиваемым артефактом** (`present_files` → `/mnt/user-data/outputs/…`) + публикуй **sha256**; Иван кладёт файл и сверяет хеш. **Не забывай вызвать `present_files`** — без него ссылок на скачивание не появится.
  - **Коммиты — только флагами `-m`** (никогда `-F -` / heredoc). Сообщения коммитов — ASCII (транслит).
- **Язык** — русский по умолчанию; идентификаторы/код/коммиты/комментарии — английский.
