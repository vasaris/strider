## 1. НЕРУШИМЫЕ ПРИНЦИПЫ (constitution проекта)

1. **Механика не живёт в LLM.** Броски/проверки/таблицы/состояние — детерминированный код. Нарративный слой (Stage 2) получит структурный результат и не имеет права выдумывать числа/исходы.
2. **Оракулы — механика движка.** Таблицы бросает rules engine; LLM интерпретирует выпавшее.
3. **`engine/` НЕ содержит контента ВК.** Ни строки правил/текста, **ни одного книжного числа** в коде. Весь контент и все числа — из `content-packs/kv/` через verified-JSON и манифест. Зеркало content-дисциплины: движок не дублирует числа пака.
4. **RNG сидируемый.** Каждый бросок воспроизводим; юнит-тесты на фиксированных сидах.
5. **Загрузчик отказывает на неверифицированном паке** (`all_verified:false` → `PackNotVerifiedError`; плюс per-file проверка).
6. **Юр-контур приватный.** Никогда не предлагай публичные анонсы, открытые ссылки, монетизацию, демо-на-фестивалях для ВК-контента.

**Правило ВК — по паку, не по памяти.** Перед любым дизайн-решением читай дословный текст карты из verified-пака. Не помнишь точно — найди в паке/PDF, не выдумывай.

**Инварианты `engine/src/`:** только ASCII (ноль кириллицы), английские идентификаторы/комментарии, ноль рантайм-импортов из `content-packs/`, ноль книжных числовых литералов в формулах.

### Ключевые паттерны, которые НЕ переоткрывать
- **Configs-from-pack:** каждая подсистема извлекает свои числа из verified-карт (`diceConfigFromPack`, `checkConfigFromPack`, `eyeConfigFromPack`, `oraclesFromPack`, `deriveConditionsConfig`, `journeyConfigsFromPack`). Числа из строк пака парсятся (regex), не зашиваются. **Бой делает так же.**
- **`HeroState`** (`src/hero/state.ts`) — фундамент, его потребляют все. Расширяй ЕГО под бой (раны/Выносливость уже есть: `endurance{current,max}`, `wounded`, `loadGear`, `shadow`).
- **Мульти-op интерпретатор эффектов** (`src/journey/effects.ts`) — единая точка применения `Effect{op,…}` к срезам, с кросс-триггерами (Тень вне боя → Око). Бой добавит свои op (урон, раны, Ненависть) — расширяй интерпретатор или сделай боевой аналог; кросс-триггер Око: **в бою Око НЕ растёт от Кости испытания** (только вне боя) — но неудача-Око в бою имеет свои последствия, читай карту.
- **`evaluateCheck(roll, tn, cfg, conditions?)`** уже умеет weary/miserable/руну. Атаки в бою — это проверки боевого навыка; переиспользуй.
- **`FeatDieFace` — размеченный union-объект** (`{kind:"number",value}` / `{kind:"eye"}` / `{kind:"gandalf_rune"}`), НЕ литерал. При ручной сборке `DiceRoll` в тестах нужен `featCandidates` и `physicalFace`.
- **Соло-коллапс ролей:** один герой играет все роли; «выбор цели» КВ-партии схлопывается. В бою аналогично: игрок ведёт и героя, и врагов (враги — через таблицы/непрозрачные эффекты, ИдО).

## 3. Текущая задача: БОЙ (combat 9 + adversaries 7)

- Соло-ведение боя — искать в ИдО (`content-packs/kv/tables/solo/*` и solo rule_cards): таблицы поведения/целей врага, как игрок отыгрывает противника без Хранителя.

**Первый шаг сессии — РАЗВЕДКА пака (как всегда), не код:** прочитать `combat.*` params (последовательность раунда, стойки/боевые задачи, как считается атака: навык+характеристика vs ЦЧ от Защиты/Парирования врага, урон/Пронзающий удар, защита/броня, Выносливость врага и Ненависть), `adversaries.format_opisaniya` (поля стат-блока врага), и solo-таблицы ведения врага. Затем — скелет.

**Бой — крупнейший кусок, многотактовый.** Предлагаемое (не догма — уточни по разведке) дробление:
1. Боевое состояние + стат-блок врага + стойки/боевые задачи (модификаторы атаки/защиты).
2. Последовательность раунда + совершение атаки (проверка боевого навыка → попадание → урон).
3. Урон/Выносливость/Ранения (Травма → Ранение → риск выбытия; защита/броня; Пронзающий удар по руне/знаку).
4. Противники: стат-блоки, Ненависть, соло-ведение целей/поведения через таблицы.
5. Выход из боя + интеграция (боевая сцена как событие, аналог журней-сцены; эффекты в общий интерпретатор).

**Зацепки из готового:** раны/Выносливость уже в `HeroState`; усталость/несчастье/подавленность уже модифицируют проверки (атаки в бою их учитывают); `removeFatigue`/восстановление есть; Око: **в бою не растёт от Кости испытания** — свериться с `eye_of_mordor` engine_note.
