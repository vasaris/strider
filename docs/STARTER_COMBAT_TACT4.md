## 2. Решения тактов 1–3 — НЕ переоткрывать, не противоречить

- **Раны персистентны на `HeroState`** (развилка такта 3 решена как A): `wounded` (метка, уже драйвит восстановление в conditions), `wound: WoundState|null` (severity/healingDays/marked), `dying`, `dead`, `permanentInjuryMarks`. Боевой слой пишет это патчами; Отдых/conditions читает.
- **Тяжесть** структурна (руна→Лёгкая, число→Серьёзная(дни=числу), Око→Ужасная→умирающий). Все числа (первая помощь 1/+1/мин1/retry1, умирающий +10/оживание 1, Укол cap 10, парир/феат по группам, +1 двуручн.) — из пака, ноль литералов.
- **Пронзающий симметричен** (`resolvePiercing(defenderSide, armourDice, injuryTn, conditions?, dice, rng)`); защитник читает феат по своей стороне; защита ДО усталости — через входные conditions. Особый урон — **только герой** (враг-особый-урон непрозрачен, keeper/oracle).
- **Особый урон — данные, не state** (`applySpecialDamage` чист): добавочный урон/раунд-баффы применяет боевой цикл (такт 5).
- **`CombatConfigs`** теперь содержит `combat/attack/wounds/specialDamage/check/conditions/dice`. Резолверы берут агрегат, в пак не лезут.
- Инвариант: ноль кириллицы в `src` (кроме предсуществующего `eye/types.ts:4` — known-minor, чистить отдельным микрофиксом), ноль импортов пака, ноль книжных литералов.
