# ADR-001: auto-verify v1 заменяет ручную сверку для verified: true

Дата: 2026-06-11. Статус: принято (Иван, явное подтверждение в чате).

## Решение
Поправка нерушимого принципа №5 и §3.3 архитектурного документа:
«verified: true = прохождение auto-verify v1, ручная сверка отменена».

## auto-verify v1 — три гейта, все зелёные:
1. **text-fidelity** — tools/extraction/independent_check.py: каждая текстовая
   ячейка JSON присутствует дословно в текстовом слое заявленных страниц
   (независимый код-путь от парсеров сборки).
2. **vision-sweep** — свежая Claude-сессия сверяет каждый JSON поячеечно со
   СКАНАМИ страниц (независимая модальность); вердикт CONFIRMED/MISMATCH
   на ячейку, включая позиции рун и числа.
3. **lynn-review PASS** — независимый review-гейт по docs/lynn_review_brief_*.

verified: true проставляется скриптом только при 1∧2∧3; в файл пишется блок
verification: {method: "auto-v1", date, gates}. Иван разбирает только MISMATCH.

## Осознанный остаточный риск
Vision ошибается на мелком кегле (прецедент: KNOWN_ISSUES F5). Размен принят
для приватного контура; ошибки, всплывшие в игре, чинятся точечно с инцидент-
записью в known_issues.

## Порядок применения к паку сессии 1
vision-sweep → разбор MISMATCH → lynn-review финального состояния → простановка.
(lynn-review идёт ПОСЛЕДНИМ: он должен видеть то, что станет verified.)
