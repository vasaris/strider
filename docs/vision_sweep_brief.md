# vision-sweep — бриф (гейт 2 протокола auto-verify v1, ADR-001)

TASK: поячеечная сверка 25 JSON-таблиц пака `content-packs/kv/tables/solo/` со СКАНАМИ
страниц книги «Игра для одного».
MODE: VERIFY ONLY. Ничего не менять, не чинить, не коммитить. Найденное расхождение —
зафиксировать, не исправлять. Не доверять ни отчётам build-сессии, ни текстовому слою
(`tools/extraction/source_pages/*.txt`) — **истина = скан**. Текстовый слой в этой проверке
не открывать вообще: он и был источником извлечения, сверка с ним циркулярна.

## Где что лежит

- JSON-файлы — в приложенном архиве, `content-packs/kv/tables/solo/*.json`.
- Сканы: файл проекта `/mnt/project/Игра_для_одного_на_рассылку_03_04_2026.pdf` — это
  ZIP-бандл; распакуй (`unzip`) и получишь `N.jpeg` (страница N, 980×1232). Смотри их
  инструментом view. Страницы для каждого файла — в его `source.pages`; сверяй по
  ПЕРВОЙ странице списка (тело книги — канон), остальные — источник диффов, их сверять
  не нужно.

## Что проверять (по типам)

Для каждого файла — каждую ячейку payload против скана:

- **oracle_yes_no**: 5 порогов («1 или больше» … «10»), подписи вероятностей, обе строки
  рун (руна = «да», Око = «нет») — скан стр. 12.
- **lore_table**: 12 секций × 6 строк × 3 слова. Критично: какая секция помечена глифом
  Ока (стилизованный глаз), какая — руной Гэндальфа (символ «ᚷ»-вида); затем выборочно
  минимум 2 полные секции дословно + по 1 случайной строке в каждой из остальных десяти.
- **feat_die_event_table** (luck, misfortune, detection_scenes): порядок — глиф Ока у
  ПЕРВОЙ строки, руны у ПОСЛЕДНЕЙ; все 12 текстов; числа в строках Ока (−1 / +2).
- **journey_scenes_table**: диапазоны Кости испытания (Око, 1, 2–3, 4–7, 8–9, 10, руна),
  тексты последствий, колонка Изнурения (3/2/2/2/1/1/—), знаки структурированных
  эффектов (`wound`, `shadow_points 2/1`, `journey_days ±1`, `hope 1`).
- **scene_detail_table** (7 шт.): 6 строк «сцена + результат», навык в КАПИТЕЛИ
  соответствует полю `skill` (ОХОТА=hunting, ИССЛЕДОВАНИЕ=exploration,
  БДИТЕЛЬНОСТЬ=awareness), подвал: текст последствий + баллы Изнурения.
- **patron_tasks_table** (6 шт.): имя покровителя в заголовке, 6 заданий дословно.
- **lookup_table / rule_parameters**: каждое ЧИСЛО против скана (вехи: баллы; пороги
  18/16/14; модификаторы +4/+2/−2/−4; Тень 1/2/3; опыт 15; братство 3; ЦЧ 18);
  тексты — дословность.

## Дисциплина UNREADABLE

Если ячейка на скане мелкая/нечитаемая при 980×1232 — вердикт `UNREADABLE`, не
CONFIRMED «на глаз». Молчаливое подтверждение нечитаемого — главный известный
failure-mode этой проверки (см. KNOWN_ISSUES F5 в архиве).

## Выход

Файл `vision_sweep_report.json`:

```json
{
  "method": "vision-sweep auto-v1", "date": "YYYY-MM-DD",
  "files": [
    {
      "id": "kv.solo.lore",
      "scan_pages_viewed": [13, 14],
      "cells_checked": 78,
      "verdicts": [
        {"cell": "sections[0].face", "verdict": "CONFIRMED"},
        {"cell": "sections[2].rows[4].focus", "verdict": "MISMATCH",
         "expected_from_scan": "…", "found_in_json": "…"}
      ],
      "summary": "CONFIRMED"
    }
  ],
  "totals": {"files_clean": 0, "files_with_mismatch": 0, "cells_unreadable": 0}
}
```

`verdicts` перечисляет ВСЕ не-CONFIRMED ячейки поимённо (JSON-path) + по одной записи
`"cell": "ALL_OTHER", "verdict": "CONFIRMED"` на файл, если остальное чисто. Гейт зелёный
при нуле MISMATCH и нуле UNREADABLE по всем 25 файлам; иначе — список на разбор Ивану.

В конце — короткая текстовая сводка: что чисто, что нет, где пришлось щуриться.

## Конвенция инлайн-глифов в прозе

В прозаических полях (`oracle_rule`, `growth_triggers[].text`, `spend_rule_text`,
`precedence`) инлайн-глифы книги, утраченные в наборе, транскрибированы как `[Око]`,
`[руна]`, `[успеха]`. На скане в этом месте стоит соответствующий символ — это CONFIRMED,
не MISMATCH. Поля `engine_note` — аннотации движка, со сканом НЕ сверяются.
