# ADR-002: гейты auto-verify для книги правил (без сканов в knowledge)

Дата: 2026-06-11. Статус: принято (Иван: «А+С давай да», «Да, ок»).
Дополняет ADR-001 для контента из kv_core (mechanics/, adversaries/).

## Контекст
КВ в knowledge — плоский текст без постраничных сканов. Vision-гейт ADR-001
в исходном виде невозможен. Главный риск книги правил — не порча слоя,
а интерпретация при структурировании карточек.

## Гейты для kv_core
1. **text-fidelity** — source_text, описания и тексты способностей дословно
   (после нормализации пробелов) присутствуют в слое kv_core; цитаты ВЫРЕЗАЮТСЯ
   из слоя по якорным парам, не перепечатываются — дословность по построению.
2a. **semantic-sweep** (свежая сессия, без PDF) — на карточку: SUPPORTED /
   PARTIAL / UNSUPPORTED — следуют ли summary и parameters из source_text
   и нет ли добавлений сверх цитат.
2b. **vision-spotcheck** (свежая сессия + нарезка PDF от Ивана,
   pikepdf для снятия RC4) — ВСЕ числа parameters/статблоков и табличные
   данные против страниц. Проза со сканами не сверяется.
3. **lynn-review** — как в ADR-001. Простановка mark_verified --dir.

## Принятые риски
- Дефект слоя в прозе (вне чисел) не ловится; типографическая разрядка
  слоя сохраняется в цитатах как есть.
- Страничная привязка — по рубрикатору, точность ±1 страница на стыках.
